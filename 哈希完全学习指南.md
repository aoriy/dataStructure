# 第一部分：哈希核心概念

## 1.1 什么是哈希

### 定义

**哈希（Hash）**是一种将任意长度的输入数据映射为固定长度输出的函数。这个输出称为**哈希值（Hash Value）**或**散列值**。

### 核心思想

> **"用一个函数把复杂的数据变成一个简单的数字，用这个数字作为索引快速找到数据"**

### 直觉理解

想象一个图书馆：
- **传统方法**：按书名字母顺序排列，查找需要逐个比对 → O(n)
- **哈希方法**：用一个公式把书名变成一个数字（如书架编号），直接去那个书架找 → O(1)

### 哈希的本质作用

| 作用 | 说明 |
|------|------|
| **快速查找** | 用哈希值作为索引，实现 O(1) 查找 |
| **数据去重** | 相同数据的哈希值相同，快速判断是否重复 |
| **数据校验** | 比对哈希值判断数据是否被篡改 |
| **负载均衡** | 用哈希将请求分配到不同服务器 |

## 1.2 希表的工作原理

### 哈希表（Hash Table）的结构

哈希表本质上是一个**数组 + 哈希函数**的组合：

```
┌─────────────────────────────────────────────┐
│  Key  →  Hash Function  →  Hash Value       │
│  "apple"  →  hash("apple")  →  3            │
│                                             │
│  Array (Buckets):                           │
│  Index:  0    1    2    3    4    5    6    │
│  Value:  -    -    -   "apple"  -    -    - │
└─────────────────────────────────────────────┘
```

### 工作流程

```
1. 插入 "apple"：
   - 计算 hash("apple") = 3
   - 在数组索引 3 处存储 "apple"

2. 查找 "apple"：
   - 计算 hash("apple") = 3
   - 直接访问数组索引 3 → 找到 "apple"

3. 删除 "apple"：
   - 计算 hash("apple") = 3
   - 将数组索引 3 处标记为空
```

### 为什么哈希表查找是 O(1)？

> **因为哈希值直接告诉我们数据在数组的哪个位置，不需要逐个遍历。**

理想情况下：
- 计算 hash(key)：O(1)
- 访问数组[hash(key)]：O(1)
- 总时间：O(1)

## 1.3 哈希函数的设计

### 好的哈希函数的标准

| 标准 | 说明 |
|------|------|
| **确定性** | 相同输入必须产生相同输出 |
| **均匀分布** | 不同输入的输出应均匀分布在哈希空间 |
| **高效计算** | 计算哈希值的时间要短 |
| **最小碰撞** | 不同输入尽量产生不同输出 |

### 常见哈希函数

#### 1. 直接定址法

```python
hash(key) = key  # 或 key mod table_size
```

**适用**：key 本身就是整数且范围可控

#### 2. 除留余数法（最常用）

```python
hash(key) = key % table_size
```

**关键**：table_size 通常选择**质数**，以减少碰撞

#### 3. 数字分析法

从 key 中抽取若干位作为哈希值，如取中间几位。

#### 4. 平方取中法

```python
hash(key) = middle_digits(key²)
```

先平方，再取中间几位。

#### 5. 字符串哈希函数

**经典方法**：将字符串视为一个大整数

```python
def string_hash(s, table_size):
    hash_value = 0
    for char in s:
        hash_value = (hash_value * 31 + ord(char)) % table_size
    return hash_value
```

**为什么用 31？**
- 31 是质数
- 31 = 2⁵ - 1，可以用位运算优化：`(hash << 5) - hash`
- Java、Python 等语言的标准库都用类似方法

#### 6. MD5 / SHA 系列（加密哈希）

| 算法 | 输出长度 | 用途 |
|------|---------|------|
| MD5 | 128 位 | 文件校验（已被破解，不推荐用于安全） |
| SHA-1 | 160 位 | 已被破解，不推荐 |
| SHA-256 | 256 位 | 加密、区块链、数字签名 |
| SHA-512 | 512 位 | 高安全性场景 |

> **注意**：加密哈希函数（SHA 系列）计算较慢，不适合用于哈希表的快速查找。哈希表通常使用简单的非加密哈希函数。

## 1.4 哈希冲突及解决方法

### 什么是哈希冲突？

当**两个不同的 key 计算出相同的哈希值**时，就发生了哈希冲突。

```
hash("apple") = 3
hash("banana") = 3  ← 冲突！两个不同的 key 映射到同一个位置
```

> **哈希冲突是不可避免的**，因为哈希表的容量有限，而可能的 key 无限。

### 冲突解决方法

#### 方法一：链地址法（Separate Chaining）—— 最常用

**原理**：每个数组位置存储一个**链表**，冲突的元素都挂在同一个链表上。

```
Array:
Index:  0        1        2        3        4
        ↓        ↓        ↓        ↓        ↓
       null     null     null    [apple]   null
                          ↓
                        [banana]
                          ↓
                        [cherry]
```

**优点**：
- 实现简单
- 不需要预先知道数据量
- 删除操作简单

**缺点**：
- 链表过长时查找效率下降
- 需要额外的链表存储空间

**代码实现**：

```python
class HashTableChaining:
    def __init__(self, size=10):
        self.size = size
        self.table = [[] for _ in range(size)]  # 每个位置是一个链表

    def hash(self, key):
        return hash(key) % self.size

    def insert(self, key, value):
        idx = self.hash(key)
        # 检查是否已存在，更新或添加
        for i, (k, v) in enumerate(self.table[idx]):
            if k == key:
                self.table[idx][i] = (key, value)
                return
        self.table[idx].append((key, value))

    def get(self, key):
        idx = self.hash(key)
        for k, v in self.table[idx]:
            if k == key:
                return v
        return None

    def delete(self, key):
        idx = self.hash(key)
        for i, (k, v) in enumerate(self.table[idx]):
            if k == key:
                self.table[idx].pop(i)
                return True
        return False
```

#### 方法二：开放地址法（Open Addressing）

**原理**：冲突时，在数组中寻找下一个空位置。

##### 线性探测（Linear Probing）

```
hash(key) 冲突 → 尝试 (hash(key) + 1) % size
                → 尝试 (hash(key) + 2) % size
                → ... 直到找到空位置
```

```
插入 "apple"：hash("apple") = 3 → 索引 3 已有数据
              尝试索引 4 → 空 → 插入成功

插入 "banana"：hash("banana") = 3 → 索引 3 已有数据
               尝试索引 4 → 已有数据
               尝试索引 5 → 空 → 插入成功
```

**优点**：
- 不需要额外的链表空间
- 缓存友好（连续访问）

**缺点**：
- 容易产生**聚集现象**（连续位置被占满）
- 删除操作复杂（需要标记为"已删除"而非"空"）

##### 二次探测（Quadratic Probing）

```
尝试序列：(hash(key) + 1²) % size
         (hash(key) + 2²) % size
         (hash(key) + 3²) % size
         ...
```

减少聚集现象。

##### 双重哈希（Double Hashing）

```
使用两个哈希函数：
step = hash2(key)
尝试序列：(hash1(key) + step) % size
         (hash1(key) + 2*step) % size
         ...
```

#### 方法三：再哈希法（Rehashing）

使用多个不同的哈希函数，直到找到空位置。

### 冲突解决方法对比

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **链地址法** | 实现简单、删除方便 | 链表过长影响效率 | 通用场景，最常用 |
| **线性探测** | 缓存友好、无额外空间 | 聪明聚集问题 | 数据量可控时 |
| **二次探测** | 减少聚集 | 可能无法找到空位 | 中等数据量 |
| **双重哈希** | 分布均匀 | 计算开销大 | 高性能要求场景 |

## 1.5 哈希表的关键指标

### 负载因子（Load Factor）

```
负载因子 = 元素数量 / 哈希表容量
         = n / m
```

| 负载因子 | 影响 |
|---------|------|
| **< 0.5** | 空间浪费，但查找很快 |
| **0.5 ~ 0.75** | 平衡空间与效率，推荐范围 |
| **> 0.75** | 冲突增加，查找效率下降，需要扩容 |

### 扩容（Rehashing）

当负载因子超过阈值时，需要**扩容并重新哈希**：

```
1. 创建更大的新数组（通常是 2 倍）
2. 重新计算所有元素的哈希值
3. 将所有元素迁移到新数组
```

**扩容代价**：O(n)，但均摊到每次插入是 O(1)

### 时间复杂度分析

| 操作 | 平均情况 | 最坏情况（所有元素冲突） |
|------|---------|----------------------|
| **查找** | O(1) | O(n) |
| **插入** | O(1) | O(n) |
| **删除** | O(1) | O(n) |

> **最坏情况极少发生**，只要哈希函数设计得当、负载因子合理，平均就是 O(1)。

# 第二部分：哈希表实现与操作

## 2.1 哈希表的基本操作

### 插入（Insert）

```python
def insert(key, value):
    idx = hash(key) % table_size
    # 处理冲突（链地址法）
    table[idx].append((key, value))
```

### 查找（Get）

```python
def get(key):
    idx = hash(key) % table_size
    for k, v in table[idx]:
        if k == key:
            return v
    return None  # 或抛出异常
```

### 删除（Delete）

```python
def delete(key):
    idx = hash(key) % table_size
    for i, (k, v) in enumerate(table[idx]):
        if k == key:
            table[idx].pop(i)
            return True
    return False
```

### 更新（Update）

```python
def update(key, new_value):
    idx = hash(key) % table_size
    for i, (k, v) in enumerate(table[idx]):
        if k == key:
            table[idx][i] = (key, new_value)
            return True
    return False  # key 不存在
```

## 2.2 不同语言的哈希表实现

### Python：dict

```python
# 创建
d = {}  # 或 d = dict()

# 插入/更新
d["apple"] = 5
d["banana"] = 3

# 查找
value = d.get("apple")  # 返回 5，不存在返回 None
value = d["apple"]      # 返回 5，不存在抛出 KeyError

# 删除
d.pop("apple")          # 删除并返回值，不存在抛出 KeyError
del d["apple"]          # 删除，不存在抛出 KeyError

# 检查存在
"apple" in d            # True 或 False

# 遍历
for key in d:           # 遍历 key
for key, value in d.items():  # 遍历 key-value
for value in d.values():      # 遍历 value
```

**Python dict 特点**：
- 使用链地址法处理冲突
- 自动扩容（负载因子约 0.66）
- key 必须是**可哈希的**（不可变类型：str, int, tuple, frozenset）
- 从 Python 3.7+ 开始，dict 保持插入顺序

### Java：HashMap

```java
// 创建
HashMap<String, Integer> map = new HashMap<>();

// 插入/更新
map.put("apple", 5);
map.put("banana", 3);

// 查找
Integer value = map.get("apple");  // 返回 5，不存在返回 null
Integer value = map.getOrDefault("apple", 0);  // 不存在返回默认值

// 删除
map.remove("apple");

// 检查存在
map.containsKey("apple");  // True 或 False
map.containsValue(5);      // 检查值是否存在

// 遍历
for (String key : map.keySet()) {
    System.out.println(key + ": " + map.get(key));
}
```

**Java HashMap 特点**：
- 使用链地址法（Java 8+ 当链表过长时转为红黑树）
- 默认初始容量 16，负载因子 0.75
- 扩容时容量翻倍

### C++：unordered_map

```cpp
// 创建
unordered_map<string, int> map;

// 插入/更新
map["apple"] = 5;
map.insert({"banana", 3});
map.emplace("cherry", 2);  // 更高效

// 查找
int value = map["apple"];  // 不存在会自动创建
auto it = map.find("apple");  // 返回迭代器，不存在返回 map.end()

// 删除
map.erase("apple");

// 检查存在
map.count("apple") > 0;  // 返回 0 或 1
map.find("apple") != map.end();

// 遍历
for (auto& [key, value] : map) {
    cout << key << ": " << value << endl;
}
```

**C++ unordered_map 特点**：
- 使用链地址法
- 平均 O(1)，最坏 O(n)
- 不保证顺序

### 三种语言对比

| 特性 | Python dict | Java HashMap | C++ unordered_map |
|------|------------|--------------|-------------------|
| **冲突处理** | 链地址法 | 链地址法 + 红黑树 | 铿地址法 |
| **默认负载因子** | ~0.66 | 0.75 | 1.0 |
| **保持顺序** | 3.7+ 是 | 否（用 LinkedHashMap） | 否 |
| **线程安全** | 否 | 否（用 ConcurrentHashMap） | 否 |

## 2.3 手写哈希表实现

### 简化版哈希表（链地址法）

```python
class SimpleHashTable:
    def __init__(self, capacity=16):
        self.capacity = capacity
        self.size = 0
        self.buckets = [[] for _ in range(capacity)]
        self.load_factor_threshold = 0.75

    def _hash(self, key):
        return hash(key) % self.capacity

    def put(self, key, value):
        if self.size / self.capacity >= self.load_factor_threshold:
            self._resize()

        idx = self._hash(key)
        bucket = self.buckets[idx]

        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)  # 更新
                return

        bucket.append((key, value))
        self.size += 1

    def get(self, key):
        idx = self._hash(key)
        bucket = self.buckets[idx]

        for k, v in bucket:
            if k == key:
                return v
        return None

    def remove(self, key):
        idx = self._hash(key)
        bucket = self.buckets[idx]

        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket.pop(i)
                self.size -= 1
                return True
        return False

    def _resize(self):
        new_capacity = self.capacity * 2
        new_buckets = [[] for _ in range(new_capacity)]

        for bucket in self.buckets:
            for key, value in bucket:
                new_idx = hash(key) % new_capacity
                new_buckets[new_idx].append((key, value))

        self.capacity = new_capacity
        self.buckets = new_buckets

    def __str__(self):
        result = []
        for i, bucket in enumerate(self.buckets):
            if bucket:
                result.append(f"Bucket {i}: {bucket}")
        return "\n".join(result)


# 使用示例
ht = SimpleHashTable()
ht.put("apple", 5)
ht.put("banana", 3)
ht.put("cherry", 7)
print(ht.get("apple"))  # 5
ht.remove("banana")
print(ht.get("banana"))  # None
```

# 第三部分：哈希的应用场景

## 3.1 六大核心应用场景

### 1. 快速查找

**场景**：判断某个元素是否在集合中

```python
# 传统方法：遍历列表 O(n)
def find(lst, target):
    for item in lst:
        if item == target:
            return True
    return False

# 哈希方法：使用 set O(1)
def find_hash(lst, target):
    s = set(lst)
    return target in s
```

### 2. 数据去重

**场景**：去除重复元素

```python
# 使用 set 去重
def deduplicate(lst):
    return list(set(lst))

# 保持顺序的去重
def deduplicate_ordered(lst):
    seen = set()
    result = []
    for item in lst:
        if item not in seen:
            seen.add(item)
            result.append(item)
    return result
```

### 3. 计数统计

**场景**：统计每个元素出现的次数

```python
from collections import Counter

# 方法一：使用 Counter
def count_elements(lst):
    return Counter(lst)

# 方法二：手动实现
def count_manual(lst):
    counts = {}
    for item in lst:
        counts[item] = counts.get(item, 0) + 1
    return counts
```

### 4. 两数之和 / 配对查找

**场景**：在数组中找到两个数满足特定条件

```python
def two_sum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []
```

### 5. 字符串匹配 / Rabin-Karp

**场景**：快速判断字符串是否匹配

```python
def rabin_karp(text, pattern):
    n, m = len(text), len(pattern)
    if m > n:
        return -1

    # 计算模式串的哈希值
    pattern_hash = hash(pattern)

    for i in range(n - m + 1):
        substring = text[i:i+m]
        if hash(substring) == pattern_hash:
            if substring == pattern:  # 确认匹配
                return i
    return -1
```

### 6. 缓存实现（LRU/LFU）

**场景**：实现缓存机制

```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = OrderedDict()

    def get(self, key):
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)  # 移到最后（最近使用）
        return self.cache[key]

    def put(self, key, value):
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.capacity:
            self.cache.popitem(last=False)  # 删除最久未使用
```

## 3.2 哈希在算法中的经典用法

### 用法一：哈希 + 数组索引映射

将复杂数据映射为数组索引，实现快速访问。

```python
# 字母异位词分组
def group_anagrams(strs):
    groups = {}
    for s in strs:
        key = ''.join(sorted(s))  # 排序后的字符串作为 key
        if key not in groups:
            groups[key] = []
        groups[key].append(s)
    return list(groups.values())
```

### 用法二：哈希 + 滑动窗口

在滑动窗口中快速判断元素是否满足条件。

```python
# 无重复字符的最长子串
def length_of_longest_substring(s):
    char_index = {}  # 记录每个字符最后出现的位置
    left = 0
    max_len = 0

    for right, char in enumerate(s):
        if char in char_index and char_index[char] >= left:
            left = char_index[char] + 1  # 移动左边界
        char_index[char] = right
        max_len = max(max_len, right - left + 1)

    return max_len
```

### 用法三：哈希 + 前缀和

用哈希记录前缀和，快速找到满足条件的子数组。

```python
# 和为 K 的子数组
def subarray_sum(nums, k):
    count = 0
    prefix_sum = 0
    prefix_count = {0: 1}  # 前缀和为 0 出现 1 次

    for num in nums:
        prefix_sum += num
        if prefix_sum - k in prefix_count:
            count += prefix_count[prefix_sum - k]
        prefix_count[prefix_sum] = prefix_count.get(prefix_sum, 0) + 1

    return count
```

### 用法四：哈希 + 双指针

配合双指针解决查找问题。

```python
# 三数之和
def three_sum(nums):
    nums.sort()
    result = []

    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i-1]:
            continue

        seen = set()
        left, right = i + 1, len(nums) - 1

        while left < right:
            total = nums[i] + nums[left] + nums[right]
            if total == 0:
                result.append([nums[i], nums[left], nums[right]])
                while left < right and nums[left] == nums[left+1]:
                    left += 1
                while left < right and nums[right] == nums[right-1]:
                    right -= 1
                left += 1
                right -= 1
            elif total < 0:
                left += 1
            else:
                right -= 1

    return result
```

### 用法五：哈希 + 位运算

用哈希记录状态，配合位运算优化。

```python
# 找出数组中只出现一次的数字（其他数字出现两次）
def single_number(nums):
    # 方法一：哈希计数
    counts = {}
    for num in nums:
        counts[num] = counts.get(num, 0) + 1
    for num, count in counts.items():
        if count == 1:
            return num

    # 方法二：位运算（更优）
    result = 0
    for num in nums:
        result ^= num
    return result
```

```python
class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = OrderedDict()

    def get(self, key):
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key, value):
        if key in self.cache:

        self.cache[key] = value
```

```python
    from collections import Counter

    need = Counter(t)
    missing = len(t)
    left = 0
    start, end = 0, float('inf')

    for right, char in enumerate(s):
        if need[char] > 0:
            missing -= 1
        need[char] -= 1

        if missing == 0:
            while left < right and need[s[left]] < 0:
                need[s[left]] += 1
                left += 1

            if right - left < end - start:
                start, end = left, right

            need[s[left]] += 1
            missing += 1
            left += 1

    return s[start:end+1] if end != float('inf') else ""
```

## 5.1 常见陷阱与易错点

### 陷阱一：不可哈希的类型作为 key

```python
# ❌ 错误：列表不能作为 key
d = {}
d[[1, 2]] = 3  # TypeError: unhashable type: 'list'

# ✅ 正确：使用 tuple
d[(1, 2)] = 3

# ✅ 正确：使用 frozenset
d[frozenset({1, 2})] = 3
```

### 陷阱二：哈希值相等不代表值相等

```python
# 哈希碰撞示例（极端情况）
a = "a"
b = some_other_string_with_same_hash

hash(a) == hash(b)  # True（哈希碰撞）
a == b              # False（值不同）

# 所以哈希查找后必须还要比对值
```

### 陷阱三：修改 key 导致找不到

```python
# ❌ 错误：key 是可变对象，修改后找不到

d[tuple(key)] = 3  # 正确做法

# 如果 key 是自定义对象，修改其属性后可能找不到
```

### 陷阱四：遍历时修改哈希表

```python
# ❌ 错误：遍历时删除元素
d = {"a": 1, "b": 2, "c": 3}
for key in d:
    if key == "b":
        del d[key]  # RuntimeError

# ✅ 正确：先收集要删除的 key
keys_to_delete = [key for key, value in d.items() if value == 2]
for key in keys_to_delete:
    del d[key]

# ✅ 正确：使用字典推导式
d = {k: v for k, v in d.items() if v != 2}
```

## 5.2 哈希表 vs 其他数据结构

### 哈希表 vs 数组

| 对比维度 | 哈希表 | 数组 |
|---------|--------|------|
| **查找** | O(1) 平均 | O(n) 或 O(1)（已知索引） |
| **插入** | O(1) 平均 | O(n)（需要移动） |
| **删除** | O(1) 平均 | O(n) |
| **空间** | O(n) + 冲突开销 | O(n) |
| **顺序** | 不保证（Python 3.7+ 保证） | 保证 |
| **适用 key** | 任意可哈希类型 | 整数索引 |

### 哈希表 vs 二叉搜索树（BST）

| 对比维度 | 哈希表 | BST |
|---------|--------|-----|
| **查找** | O(1) 平均，O(n) 最坏 | O(log n) |
| **插入** | O(1) 平均 | O(log n) |
| **删除** | O(1) 平均 | O(log n) |
| **有序遍历** | ❌ 不支持 | ✅ 支持 |
| **范围查询** | ❌ 不支持 | ✅ 支持 |
| **最值查询** | ❌ 不支持 | ✅ 支持 |
| **空间** | O(n) | O(n) |

> **选择原则**：
> - 只需要快速查找 → 哈希表
> - 需要有序遍历/范围查询 → BST（或 TreeMap/OrderedDict）

### 哈希表 vs 跳表（Skip List）

| 对比维度 | 哈希表 | 跳表 |
|---------|--------|-----|
| **查找** | O(1) | O(log n) |
| **有序遍历** | ❌ | ✅ |
| **实现复杂度** | ⭐⭐ | ⭐⭐⭐ |
| **并发友好** | ❌ | ✅ |

## 5.3 哈希碰撞攻击与安全

### 什么是哈希碰撞攻击？

攻击者构造大量哈希值相同的输入，使哈希表退化为链表，查找效率从 O(1) 变成 O(n)，导致服务拒绝（DoS）。

### 防护措施

| 方法 | 说明 |
|------|------|
| **随机哈希种子** | 启动时随机选择哈希函数参数，攻击者无法预测 |
| **限制输入大小** | 控制哈希表的元素数量上限 |
| **使用安全哈希** | 如 SipHash，专门设计用于防止碰撞攻击 |
| **监控异常** | 检测哈希表操作时间异常增长 |

> **Python 3.3+** 默认启用哈希随机化，防止碰撞攻击。

## 哈希题型识别指南

### 高频信号词

| 信号词 | 可能的哈希应用 |
|--------|--------------|
| "是否存在" / "查找" | 哈希快速查找 |
| "重复" / "唯一" | 哈希去重 |
| "计数" / "频率" | 哈希计数 |
| "异位词" | 排序后作为 key |
| "子串" / "子数组" | 滑动窗口 + 哈希 / 前缀和 + 哈希 |
| "缓存" | 哈希 + 链表 |
| "配对" / "两数之和" | 哈希查找 complement |

### 判断流程

```
题目涉及"查找"或"判断存在"？
├── 是 → 考虑哈希
└── 否
    ├── 题目涉及"计数"或"频率"？
    │   ├── 是 → 哈希计数
    │   └── 否
    │       ├── 题目涉及"去重"或"唯一"？
    │       │   ├── 是 → 使用 set
    │       │   └── 否
    │       │       ├── 题目涉及"子数组/子串"？
    │       │       │   ├── 是 → 滑动窗口/前缀和 + 哈希
    │       │       │   └── 否 → 考虑其他方法
```
