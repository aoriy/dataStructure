## 一、问题建模与本质分析

### 1.1 将业务问题转化为图论问题

这道题的核心是**将告警关联关系建模为有向图**：

```
主告警 → 次告警  （有向边：从主指向次）
```

每一条输入 `a b` 表示：`a` 是主告警，`b` 是次告警，即存在一条有向边 `a → b`。

**示例**：输入 `a b` 和 `c b`

```
  a ──→ b ←── c
```

这表示 `b` 同时被 `a` 和 `c` 作为次告警关联。

### 1.2 两种检测目标的图论含义

| 业务情况 | 图论含义 | 检测方法 |
|---------|---------|---------|
| **情况1**：一个次告警关联了多个主告警 | 有向图中某个节点有**多个入边**（入度 ≥ 2） | 统计每个节点的入边来源集合 |
| **情况2**：关联关系中存在环路 | 有向图中存在**环**（Cycle） | 拓扑排序 / DFS 环检测 |

### 1.3 输出优先级规则

```
情况1 → 校验码 1001
情况2 → 校验码 1002
都正常 → 校验码 1003
同时存在 → 取最大校验码（即 1002 优先）
```

> **关键理解**：情况2（环路）比情况1（多主关联）更严重，所以校验码更大、优先级更高。

## 二、情况1：多主关联检测

### 2.1 问题本质

**一个次告警被多个主告警关联** = 有向图中某个节点被多个不同的节点指向。

### 2.2 数据结构选择

使用 **哈希表（字典）**：`secondary_to_primary = {次告警: {主告警1, 主告警2, ...}}`

```python
secondary_to_primary = {}  # dict[str, set[str]]

# 遍历每条关联关系
for primary, secondary in relations:
    if secondary not in secondary_to_primary:
        secondary_to_primary[secondary] = set()
    secondary_to_primary[secondary].add(primary)
```

### 2.3 检测逻辑

```python
# 找出所有主告警数量 > 1 的次告警
multi_primary = []
for secondary, primaries in secondary_to_primary.items():
    if len(primaries) > 1:
        multi_primary.append(secondary)

# 按字典序升序排列
multi_primary.sort()
```

### 2.4 图解示例

**输入**：
```
a b
c b
d e
```

**构建的哈希表**：

```
secondary_to_primary = {
    "b": {"a", "c"},   ← 2个主告警，触发情况1
    "e": {"d"}          ← 1个主告警，正常
}
```

**结果**：`b` 被多个主告警关联 → 输出 `[1001,(b)]`

## 三、情况2：有向图环检测

### 3.1 问题本质

**关联关系中存在环路** = 有向图中存在一条从某个节点出发、经过若干条边后回到该节点的路径。

### 3.2 方法一：拓扑排序（Kahn 算法）—— 推荐解法

#### 核心思想

> 如果有向图存在环，那么环中的所有节点**入度永远不为 0**，无法被拓扑排序处理。

#### 算法步骤

```
1. 计算所有节点的入度
2. 将所有入度为 0 的节点入队
3. 循环：
   a. 从队列中取出一个节点
   b. 将该节点的所有邻居的入度减 1
   c. 如果某个邻居的入度变为 0，将其入队
4. 如果处理完的节点数 < 总节点数 → 存在环
```

#### 图解示例

**输入**：
```
a b
b c
c a
```

**构建的图**：

```
a → b → c → a  （构成环）
```

**拓扑排序过程**：

```
初始入度：a=1, b=1, c=1
入度为 0 的节点：无！

队列始终为空 → 无法开始处理
处理完的节点数 = 0 < 总节点数 3 → 存在环 ✅
```

#### 另一个例子（无环）

**输入**：
```
a b
b c
```

**构建的图**：

```
a → b → c
```

**拓扑排序过程**：

```
初始入度：a=0, b=1, c=1

第1轮：a 入度=0 → 出队处理 → b 的入度变为 0
第2轮：b 入度=0 → 出队处理 → c 的入度变为 0
第3轮：c 入度=0 → 出队处理

处理完的节点数 = 3 = 总节点数 3 → 无环 ✅
```

### 3.3 方法二：DFS 环检测（三色标记法）

#### 核心思想

> 在 DFS 过程中，如果遇到一个**正在当前递归路径上**的节点（灰色节点），说明存在环。

#### 三色标记

| 颜色 | 含义 |
|------|------|
| **白色（0）** | 未访问 |
| **灰色（1）** | 正在访问（在当前递归路径上） |
| **黑色（2）** | 已完成访问（不在当前路径上） |

#### 算法步骤

```
1. 所有节点初始化为白色
2. 对每个白色节点执行 DFS
3. DFS 进入节点时：标记为灰色
4. DFS 递归访问所有邻居
5. 如果邻居是灰色 → 发现环！
6. DFS 离开节点时：标记为黑色
```

#### 代码

```python
def has_cycle_dfs(graph):
    color = {}  # 0=白, 1=灰, 2=黑

    for node in graph:
        color[node] = 0

    def dfs(node):
        color[node] = 1  # 标记为灰色
        for neighbor in graph.get(node, []):
            if color[neighbor] == 1:  # 遇到灰色节点 → 有环
                return True
            if color[neighbor] == 0:
                if dfs(neighbor):
                    return True
        color[node] = 2  # 标记为黑色
        return False

    for node in graph:
        if color[node] == 0:
            if dfs(node):
                return True
    return False
```

### 3.4 两种方法对比

| 对比维度 | 拓扑排序（Kahn） | DFS 三色标记 |
|---------|-----------------|-------------|
| **核心思想** | 入度驱动，逐步消除入度为0的节点 | 递归路径检测，灰色节点回访 |
| **时间复杂度** | O(V + E) | O(V + E) |
| **空间复杂度** | O(V)（队列 + 入度数组） | O(V)（递归栈 + 颜色数组） |
| **能否输出环** | ❌ 只能判断是否存在环 | ✅ 可以记录环的路径 |
| **实现难度** | ⭐⭐ 简单直观 | ⭐⭐⭐ 稍复杂 |
| **推荐场景** | 只需判断是否有环 | 需要找出具体环路径 |

> **本题推荐拓扑排序**：因为只需要判断是否存在环，不需要输出具体路径，拓扑排序更直观。

## 四、完整代码实现

### 4.1 Python 实现

```python
from collections import defaultdict, deque

def solve():
    # 读取输入
    lines = []
    while True:
        try:
            line = input().strip()
            if line:
                lines.append(line)
        except EOFError:
            break

    relations = []
    all_nodes = set()

    for line in lines:
        parts = line.split()
        if len(parts) == 2:
            primary, secondary = parts[0], parts[1]
            relations.append((primary, secondary))
            all_nodes.add(primary)
            all_nodes.add(secondary)

    # ===== 情况1：多主关联检测 =====
    secondary_to_primary = defaultdict(set)
    for primary, secondary in relations:
        secondary_to_primary[secondary].add(primary)

    multi_primary_list = []
    for secondary, primaries in secondary_to_primary.items():
        if len(primaries) > 1:
            multi_primary_list.append(secondary)

    multi_primary_list.sort()  # 按字典序升序
    has_multi_primary = len(multi_primary_list) > 0

    # ===== 情况2：环检测（拓扑排序） =====
    # 构建邻接表和入度表
    graph = defaultdict(list)
    in_degree = defaultdict(int)

    for node in all_nodes:
        in_degree[node] = 0  # 初始化所有节点的入度

    for primary, secondary in relations:
        graph[primary].append(secondary)
        in_degree[secondary] += 1

    # Kahn 算法
    queue = deque()
    for node in all_nodes:
        if in_degree[node] == 0:
            queue.append(node)

    processed = 0
    while queue:
        node = queue.popleft()
        processed += 1
        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)

    has_cycle = processed < len(all_nodes)

    # ===== 输出结果 =====
    if has_multi_primary and has_cycle:
        # 两种情况都存在，取最大校验码
        print("[1002,cycle]")
    elif has_cycle:
        print("[1002,cycle]")
    elif has_multi_primary:
        result = ",".join(multi_primary_list)
        print(f"[1001,({result})]")
    else:
        print("[1003,Verified]")


if __name__ == "__main__":
    solve()
```

### 4.2 代码逐行解析

```python
# ──── 第1步：读取输入 ────
# 逐行读取，解析出 (主告警, 次告警) 对
# 同时收集所有出现过的节点名称

# ──── 第2步：情况1检测 ────
# secondary_to_primary: 字典，key=次告警，value=主告警集合
# 如果某个次告警的主告警集合大小 > 1，则触发情况1
# 结果按字典序排序

# ──── 第3步：情况2检测 ────
# 构建有向图的邻接表 + 入度表
# Kahn 算法：
#   - 入度为0的节点入队
#   - 不断出队、减少邻居入度
#   - 如果最终处理数 < 总节点数 → 有环

# ──── 第4步：输出 ────
# 同时存在 → 1002（优先级更高）
# 仅有情况1 → 1001
# 仅有情况2 → 1002
# 都没有 → 1003
```

### 4.3 运行验证

**样例1**：
```
输入：
a b
c b

过程：
  情况1：b 的主告警集合 = {a, c}，大小=2 → 触发
  情况2：图 a→b, c→b，无环
  输出：[1001,(b)]
```

**样例2**：
```
输入：
a b
b a

过程：
  情况1：a 的主告警集合 = {b}，大小=1；b 的主告警集合 = {a}，大小=1 → 不触发
  情况2：图 a→b→a，存在环 → 触发
  输出：[1002,cycle]
```

## 五、复杂度分析

### 5.1 时间复杂度

| 步骤 | 复杂度 | 说明 |
|------|--------|------|
| 读取输入 | O(m) | m 为关联关系数 |
| 情况1检测 | O(m) | 遍历所有关系，哈希表操作 O(1) |
| 构建图 | O(m) | 邻接表 + 入度表 |
| 拓扑排序 | O(V + E) | V 为节点数，E 为边数 |
| 排序输出 | O(k log k) | k 为多主关联的次告警数，k ≤ V |
| **总计** | **O(V + E)** | 即 O(V + m)，因为 E = m |

### 5.2 空间复杂度

| 数据结构 | 复杂度 | 说明 |
|---------|--------|------|
| 邻接表 | O(V + E) | 存储图结构 |
| 入度表 | O(V) | 每个节点一个入度值 |
| secondary_to_primary | O(V + E) | 哈希表 + 集合 |
| 队列 | O(V) | 最多存储所有节点 |
| **总计** | **O(V + E)** | |

### 5.3 数据范围适配

```
告警名称长度：1 ~ 256
关联关系数量：1 ~ 1000
节点数 V ≤ 2000（每条关系最多引入 2 个新节点）
边数 E = m ≤ 1000

O(V + E) = O(3000) → 完全可以接受 ✅
```

## 七、扩展思考

### 7.1 如果需要输出具体的环路径怎么办？

拓扑排序只能判断**是否存在环**，不能输出具体路径。如果需要输出环路径，需要改用 **DFS 三色标记法**：

```python
def find_cycle(graph):
    color = {node: 0 for node in graph}  # 0=白, 1=灰, 2=黑
    parent = {}

    def dfs(node):
        color[node] = 1  # 灰色
        for neighbor in graph.get(node, []):
            if color[neighbor] == 1:  # 找到环
                # 回溯构建环路径
                cycle = [neighbor, node]
                current = node
                while parent.get(current) != neighbor:
                    current = parent[current]
                    cycle.append(current)
                cycle.append(neighbor)
                cycle.reverse()
                return cycle
            if color[neighbor] == 0:
                parent[neighbor] = node
                result = dfs(neighbor)
                if result:
                    return result
        color[node] = 2  # 黑色
        return None

    for node in graph:
        if color[node] == 0:
            result = dfs(node)
            if result:
                return result
    return None
```

### 7.2 如果需要找出所有环怎么办？

使用 **Tarjan 强连通分量算法**（SCC）：
- 找出所有强连通分量
- 如果某个强连通分量的大小 > 1，则其中包含环
- 如果某个强连通分量大小 = 1，检查是否存在自环

### 7.3 如果关联关系数量非常大（百万级）怎么办？

- **情况1**：使用哈希表仍然高效，O(m) 时间
- **情况2**：拓扑排序仍然是 O(V + E)，但需要考虑：
  - 使用更高效的队列（如 `collections.deque`）
  - 避免递归（防止栈溢出）
  - 考虑并行化处理（将图分区）
