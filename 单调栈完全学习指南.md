# 第一部分：单调栈核心概念

## 1.1 什么是单调栈

### 定义

单调栈是一种**栈内元素保持单调递增或单调递减**的栈数据结构。在遍历数组的过程中，通过维护栈的单调性来高效地找到每个元素的**下一个更大/更小元素**。

### 核心思想

> **"维护一个有序的栈，遇到不满足单调性的元素就弹出，直到重新满足单调性为止"**

### 直觉理解

想象你站在一排人中间，向右看：
- 你能看到比你矮的人
- 但如果遇到一个比你高的人，你就**看不到**他后面的人了
- 单调栈就是帮你**高效找到"第一个挡住你视线的人"**

### 生活中的类比

**排队看身高**：一排人从左到右站立，每个人想知道右边第一个比自己高的人是谁。单调栈就像是从右往左扫描，维护一个身高递减的队伍。

---

## 1.2 单调栈的分类

| 类型 | 栈内元素顺序 | 用途 |
|------|------------|------|
| **单调递增栈** | 栈底 → 栈顶：小 → 大 | 找下一个更小元素、求柱状图最大矩形 |
| **单调递减栈** | 栈底 → 栈顶：大 → 小 | 找下一个更大元素、温度问题 |

### 图解单调递减栈的工作过程

以数组 `[2, 1, 5, 6, 2, 3]` 为例，求每个元素的**下一个更大元素**：

```
遍历过程（维护单调递减栈）：

元素 2：栈空 → 直接入栈
栈：[2]

元素 1：1 < 2，满足递减 → 直接入栈
栈：[2, 1]

元素 5：5 > 1，不满足递减 → 弹出 1，1 的下一个更大元素是 5
       5 > 2，不满足递减 → 弹出 2，2 的下一个更大元素是 5
       栈空 → 5 入栈
栈：[5]

元素 6：6 > 5，不满足递减 → 弹出 5，5 的下一个更大元素是 6
       栈空 → 6 入栈
栈：[6]

元素 2：2 < 6，满足递减 → 直接入栈
栈：[6, 2]

元素 3：3 > 2，不满足递减 → 弹出 2，2 的下一个更大元素是 3
       3 < 6，满足递减 → 3 入栈
栈：[6, 3]

遍历结束，栈中剩余元素没有下一个更大元素：
6 → -1，3 → -1

结果：[5, 5, 6, -1, 3, -1]
```

---

## 1.3 单调栈 vs 普通栈

| 对比维度 | 普通栈 | 单调栈 |
|---------|-------|--------|
| **栈内顺序** | 无特殊要求 | 严格单调递增或递减 |
| **操作特点** | 只做 push/pop | push 前可能需要先 pop 若干元素 |
| **核心操作** | LIFO 后进先出 | 弹出 → 处理 → 入栈 |
| **时间复杂度** | O(1) 每次操作 | O(n) 整体（均摊分析） |
| **典型用途** | 括号匹配、表达式求值 | 下一个更大/更小元素、柱状图 |
| **本质** | 数据存储结构 | **算法思想**（用栈实现高效搜索） |

> **关键理解**：单调栈不仅仅是一种数据结构，更是一种**算法思想**。它利用栈的单调性，将 O(n²) 的暴力搜索优化为 O(n)。

---

## 1.4 为什么单调栈能高效解决问题

### 暴力解法 vs 单调栈

**问题**：求每个元素右边第一个比它大的元素

**暴力解法**：
```python
for i in range(n):
    for j in range(i + 1, n):
        if nums[j] > nums[i]:
            result[i] = nums[j]
            break
```
- 时间复杂度：O(n²)
- 每个元素都要向右扫描

**单调栈解法**：
```python
stack = []
for num in nums:
    while stack and num > stack[-1]:
        # 弹出的元素找到了它的下一个更大元素
        stack.pop()
    stack.append(num)
```
- 时间复杂度：O(n)
- 每个元素最多入栈一次、出栈一次

### 核心原理

> **每个元素最多被 push 一次和 pop 一次，因此总操作次数是 O(n)，均摊到每个元素是 O(1)。**

单调栈之所以高效，是因为：
1. **不回溯**：一旦元素出栈，就不再需要考虑它
2. **批量处理**：一次弹出可能解决多个元素的答案
3. **信息压缩**：栈中只保留"有用的"元素（尚未找到答案的候选）

## 1.5 单调栈的遍历方向

### 两种遍历方向

| 遍历方向 | 特点 | 适用场景 |
|---------|------|---------|
| **从左到右** | 栈顶元素是"左边最近的" | 求左边第一个更大/更小元素 |
| **从右到左** | 栈顶元素是"右边最近的" | 求右边第一个更大/更小元素 |

### 选择遍历方向的规则

```
求"右边"的第一个更大/更小 → 从右到左遍历
求"左边"的第一个更大/更小 → 从左到右遍历
```

> ⚠️ 但实际上，**从左到右遍历也可以求右边第一个更大元素**（在弹出时记录答案）。两种方向都可以，选择你觉得更直观的一种即可。本指南后续模板主要使用**从左到右**的方式，因为它更常用且更灵活。

### 遍历方向与栈类型的关系

| 求解目标 | 遍历方向 | 栈的类型 | 说明 |
|---------|---------|---------|------|
| 右边第一个更大 | 从左到右 | 单调递减栈 | 弹出时记录答案 |
| 右边第一个更小 | 从左到右 | 单调递增栈 | 弹出时记录答案 |
| 左边第一个更大 | 从左到右 | 单调递减栈 | 入栈前记录答案 |
| 左边第一个更小 | 从左到右 | 单调递增栈 | 入栈前记录答案 |
| 右边第一个更大 | 从右到左 | 单调递减栈 | 入栈前记录答案 |
| 右边第一个更小 | 从右到左 | 单调递增栈 | 入栈前记录答案 |

# 第二部分：单调栈代码模板

## 2.1 通用框架

```python
def monotonic_stack(nums):
    stack = []  # 存储索引（推荐）或存储值
    result = [-1] * len(nums)  # 初始化结果

    for i, num in enumerate(nums):
        # 核心循环：不满足单调性就弹出
        while stack and 满足弹出条件(num, nums[stack[-1]]):
            top_idx = stack.pop()
            # 在弹出时处理/记录答案
            result[top_idx] = ...

        stack.append(i)

    return result
```

### 关键要点

1. **栈中存索引还是存值？**
   - ✅ **推荐存索引**：可以同时获取值和位置，更灵活
   - 存值：代码更简单，但无法获取位置信息

2. **弹出条件取决于栈的类型**：
   - 单调递减栈：`nums[i] > nums[stack[-1]]` 时弹出
   - 单调递增栈：`nums[i] < nums[stack[-1]]` 时弹出

3. **何时记录答案？**
   - 弹出时记录：求"右边"的答案
   - 入栈前记录：求"左边"的答案

## 2.2 模板一：求右边第一个更大元素

> **经典题目**：LeetCode 739 - 每日温度

```python
def next_greater_element(nums):
    """
    返回数组中每个元素右边第一个比它大的元素的值
    如果不存在则返回 -1
    """
    n = len(nums)
    result = [-1] * n
    stack = []  # 单调递减栈，存索引

    for i in range(n):
        # 当前元素比栈顶大 → 栈顶元素的答案找到了
        while stack and nums[i] > nums[stack[-1]]:
            top_idx = stack.pop()
            result[top_idx] = nums[i]
        stack.append(i)

    return result
```

**示例**：
```
输入：[2, 1, 5, 6, 2, 3]
输出：[5, 5, 6, -1, 3, -1]
解释：
  2 的右边第一个更大是 5
  1 的右边第一个更大是 5
  5 的右边第一个更大是 6
  6 的右边没有更大的 → -1
  2 的右边第一个更大是 3
  3 的右边没有更大的 → -1
```

### 变体：求右边第一个更大元素的**索引**（而非值）

```python
def next_greater_index(nums):
    n = len(nums)
    result = [-1] * n
    stack = []

    for i in range(n):
        while stack and nums[i] > nums[stack[-1]]:
            top_idx = stack.pop()
            result[top_idx] = i  # 记录索引而非值
        stack.append(i)

    return result
```

### 变体：求右边第一个更大元素的**距离**

> **LeetCode 739 - 每日温度**：求距离下一个更高温度还要等几天

```python
def daily_temperatures(temperatures):
    n = len(temperatures)
    result = [0] * n
    stack = []  # 单调递减栈

    for i in range(n):
        while stack and temperatures[i] > temperatures[stack[-1]]:
            prev_idx = stack.pop()
            result[prev_idx] = i - prev_idx  # 距离 = 当前索引 - 之前索引
        stack.append(i)

    return result
```

**示例**：
```
输入：[73, 74, 75, 71, 69, 72, 76, 73]
输出：[1, 1, 4, 2, 1, 1, 0, 0]
```

## 2.3 模板二：求左边第一个更大元素

```python
def previous_greater_element(nums):
    """
    返回数组中每个元素左边第一个比它大的元素的值
    如果不存在则返回 -1
    """
    n = len(nums)
    result = [-1] * n
    stack = []  # 单调递减栈

    for i in range(n):
        # 入栈前：栈顶就是左边第一个更大的元素
        while stack and nums[i] >= nums[stack[-1]]:
            stack.pop()
        if stack:
            result[i] = nums[stack[-1]]
        stack.append(i)

    return result
```

```
  1 的左边第一个更大是 2
  5 的左边没有更大的 → -1
  6 的左边没有更大的 → -1
  2 的左边第一个更大是 6
  3 的左边第一个更大是 6
```

## 2.6 模板五：环形数组中的下一个更大元素

> **LeetCode 503 - 下一个更大元素 II**

**技巧**：将数组**长度翻倍**（逻辑上），即遍历 `2n` 个元素，索引取模。

```python
def next_greater_circular(nums):
    """
    环形数组中每个元素的下一个更大元素
    """
    n = len(nums)
    result = [-1] * n
    stack = []  # 单调递减栈

    # 遍历两轮（0 到 2n-1），索引取模
    for i in range(2 * n):
        idx = i % n
        while stack and nums[idx] > nums[stack[-1]]:
            top_idx = stack.pop()
            result[top_idx] = nums[idx]
        stack.append(idx)

    return result
```

**示例**：
```
输入：[1, 2, 1]
输出：[2, -1, 2]
解释（环形）：
  1 的下一个更大是 2
  2 的下一个更大不存在 → -1
  1 的下一个更大是 2（绕回到开头）
```

> **注意**：虽然遍历了 2n 次，但每个元素最多入栈一次、出栈一次，时间复杂度仍为 O(n)。

## 2.7 模板六：求每个元素的"视野"范围（柱状图最大矩形）

> **LeetCode 84 - 柱状图中的最大矩形**

**核心思想**：对每个柱子，找到它能向左和向右扩展的最大范围（即左边第一个比它矮的位置和右边第一个比它矮的位置），矩形面积 = 高度 × 宽度。

```python
def largest_rectangle_area(heights):
    """
    柱状图中的最大矩形面积
    """
    n = len(heights)
    # left[i] = i 左边第一个比 heights[i] 小的元素的索引
    left = [0] * n
    # right[i] = i 右边第一个比 heights[i] 小的元素的索引
    right = [n] * n

    # 求左边第一个更小 → 单调递增栈，从左到右
    stack = []
    for i in range(n):
        while stack and heights[i] <= heights[stack[-1]]:
            stack.pop()
        left[i] = stack[-1] + 1 if stack else 0
        stack.append(i)

    # 求右边第一个更小 → 单调递增栈，从右到左
    stack = []
    for i in range(n - 1, -1, -1):
        while stack and heights[i] <= heights[stack[-1]]:
            stack.pop()
        right[i] = stack[-1] - 1 if stack else n - 1
        stack.append(i)

    # 计算最大面积
    max_area = 0
    for i in range(n):
        width = right[i] - left[i] + 1
        area = heights[i] * width
        max_area = max(max_area, area)

    return max_area
```

### 优化写法：一遍扫描 + 哨兵

```python
def largest_rectangle_area_optimized(heights):
    """
    优化版：一遍扫描，末尾添加高度 0 作为哨兵
    """
    heights.append(0)  # 哨兵，强制弹出所有元素
    stack = [-1]  # 哨兵索引，避免栈空判断
    max_area = 0

    for i in range(len(heights)):
        while stack[-1] != -1 and heights[i] < heights[stack[-1]]:
            h = heights[stack.pop()]
            w = i - stack[-1] - 1
            max_area = max(max_area, h * w)
        stack.append(i)

    heights.pop()  # 恢复数组
    return max_area
```

**示例**：
```
输入：[2, 1, 5, 6, 2, 3]
输出：10
解释：最大矩形是以高度 5 为高，宽度 2 为宽的矩形（面积 10）
```

# 第三部分：关键技巧与常见陷阱

## 3.1 六大核心技巧

### 技巧一：栈中存索引而非值

```python
# ✅ 推荐：存索引
stack = [i]  # 可以同时获取值 nums[i] 和位置 i

# ❌ 不推荐：存值
stack = [nums[i]]  # 丢失了位置信息，很多问题无法解决
```

**为什么？**
- 需要计算距离时（如每日温度），必须知道索引
- 需要同时获取值和位置时，存索引更灵活
- 处理重复元素时，索引用于区分相同值的不同位置

### 技巧二：哨兵技巧（简化边界处理）

在数组首尾添加哨兵值，避免繁琐的边界判断。

```python
# 柱状图最大矩形：栈底放 -1 作为哨兵
stack = [-1]

# 环形数组：遍历 2n 次
for i in range(2 * n):
    idx = i % n
```

**常见哨兵用法**：

| 场景 | 哨兵值 | 作用 |
|------|-------|------|
| 栈底哨兵 | `-1` | 避免栈空判断 |
| 数组末尾 | `0` 或 `-∞` | 强制弹出所有剩余元素 |
| 环形数组 | 遍历 `2n` 次 | 模拟环形遍历 |

### 技巧三：处理重复元素的等号选择

单调栈中 `>` 和 `>=` 的选择决定了**重复元素的处理方式**：

```python
# 严格单调（允许重复元素留在栈中）
while stack and nums[i] > nums[stack[-1]]:
    stack.pop()

# 非严格单调（弹出相同值的元素）
while stack and nums[i] >= nums[stack[-1]]:
    stack.pop()
```

| 等号选择 | 效果 | 适用场景 |
|--------|------|---------|
| `>` （不含等号） | 相同元素不会被弹出，留在栈中 | 需要保留重复元素信息 |
| `>=`（含等号） | 相同元素会被弹出，只保留最后一个 | 柱状图最大矩形（避免重复计算） |

> **经验法则**：如果题目中有重复元素且需要区分它们的位置，用 `>`；如果重复元素不影响结果，用 `>=` 更简洁。

### 技巧四：反向遍历技巧

有些问题从右向左遍历更自然：

```python
# 从右向左遍历，栈维护"右边的信息"
stack = []
for i in range(n - 1, -1, -1):
    while stack and nums[i] > nums[stack[-1]]:
        stack.pop()
    result[i] = stack[-1] if stack else -1
    stack.append(i)
```

**适用场景**：
- 需要快速获取"右边"的信息
- 从左到右遍历时逻辑不够直观

### 技巧五：一次遍历同时求左右边界

对于柱状图类问题，可以一次遍历同时确定左右边界：

```python
# 利用单调栈一次遍历
stack = []
for i in range(n):
    while stack and heights[i] < heights[stack[-1]]:
        mid = stack.pop()
        left = stack[-1] if stack else -1
        right = i
        width = right - left - 1
        area = heights[mid] * width
        max_area = max(max_area, area)
    stack.append(i)
```

### 技巧六：单调栈 + DP

将单调栈与动态规划结合，解决更复杂的问题：

```python
# 单调栈确定边界，DP 计算最优解
dp[i] = max(dp[i], dp[stack[-1]] + some_value)
```

**适用场景**：
- 需要在单调栈确定的范围内做进一步计算
- 柱状图扩展、最大子矩阵等问题

## 3.2 常见陷阱与易错点

### 陷阱一：遍历方向搞错

| 错误 | 正确 |
|------|------|
| 求右边更大却从右到左遍历 | 求右边更大通常从左到右遍历（弹出时记录） |
| 求左边更大却从左到右弹出 | 求左边更大在入栈前记录 |

### 陷阱二：等号方向搞反

```python
# ❌ 错误：求"更大"却用了 >=
while stack and nums[i] >= nums[stack[-1]]:  # 等号方向错误
    stack.pop()

# ✅ 正确：根据题目要求选择 > 或 >=
# 如果要求"严格更大"，用 >
# 如果允许等于也算，用 >=
```

### 陷阱三：忘记处理栈中剩余元素

```python
for i in range(n):
    while stack and nums[i] > nums[stack[-1]]:
        top_idx = stack.pop()
        result[top_idx] = nums[i]
    stack.append(i)

# ❌ 忘记：遍历结束后栈中元素没有"下一个更大"
# ✅ 正确：result 已初始化为 -1，栈中剩余元素自然保持 -1
```

> **如果 result 初始化为 -1，则不需要额外处理**。但如果初始化为其他值，需要遍历结束后清空栈。

### 陷阱四：存值而非索引导致信息丢失

```python
# ❌ 错误：存值
stack = [2, 1, 5]
# 无法知道 2 和 1 和 5 分别在数组的哪个位置

# ✅ 正确：存索引
stack = [0, 1, 2]  # nums[0]=2, nums[1]=1, nums[2]=5
```

### 陷阱五：环形数组忘记取模

```python
# ❌ 错误：直接遍历 2n
for i in range(2 * n):
    # 忘记 idx = i % n

# ✅ 正确：
for i in range(2 * n):
    idx = i % n
```

### 陷阱六：柱状图问题中宽度计算错误

```python
# 宽度 = 右边界 - 左边界 - 1
# ❌ 错误：忘记减 1
width = right - left

# ✅ 正确：
width = right - left - 1
```

## 3.3 单调栈 vs 单调队列

| 对比维度 | 单调栈 | 单调队列 |
|---------|--------|---------|
| **数据结构** | 栈（LIFO） | 队列（FIFO）+ 双端队列 |
| **操作** | 只从栈顶操作 | 可从两端操作 |
| **核心用途** | 找下一个更大/更小元素 | 滑动窗口最大/最小值 |
| **遍历方向** | 通常单向 | 窗口滑动（双向） |
| **典型题目** | 每日温度、柱状图 | 滑动窗口最大值 |
| **时间复杂度** | O(n) | O(n) |

### 何时用单调栈 vs 单调队列

```
问题涉及"下一个"更大/更小？
├── 是 → 单调栈
└── 否
    ├── 问题涉及"滑动窗口"内的最大/最小？
    │   ├── 是 → 单调队列
    │   └── 否 → 考虑其他方法
```

## 3.4 复杂度分析

### 时间复杂度

| 问题类型 | 时间复杂度 | 说明 |
|---------|-----------|------|
| 下一个更大/更小元素 | O(n) | 每个元素最多入栈出栈各一次 |
| 柱状图最大矩形 | O(n) | 一次或两次遍历 |
| 环形数组 | O(n) | 遍历 2n 但操作仍为 O(n) |
| 单调栈 + DP | O(n) ~ O(n²) | 取决于 DP 部分的复杂度 |

### 空间复杂度

| 问题类型 | 空间复杂度 | 说明 |
|---------|-----------|------|
| 基本单调栈 | O(n) | 栈最多存储 n 个元素 |
| 柱状图最大矩形 | O(n) | 栈 + 结果数组 |
| 单调栈 + DP | O(n) | 栈 + DP 数组 |

### 为什么是 O(n) 而不是 O(n²)？

**均摊分析**：
- 每个元素**最多入栈 1 次**
- 每个元素**最多出栈 1 次**
- 总操作次数 ≤ 2n = O(n)

```
内层 while 循环虽然看起来是 O(n²)，但实际上：
  所有 pop 操作的总次数 ≤ n
  因为每个元素只能被 pop 一次
```

## 4.2 入门级题目

### 🟢 LeetCode 739 - 每日温度 ⭐⭐

**难度**：中等 | **类型**：下一个更大元素（距离）

**题目**：给定一个整数数组 `temperatures`，表示每天的温度，返回一个数组 `answer`，其中 `answer[i]` 是指对于第 `i` 天，下一个更高温度出现在几天后。

**思路**：单调递减栈，弹出时计算距离。

```python
def dailyTemperatures(temperatures):
    n = len(temperatures)
    answer = [0] * n
    stack = []

    for i in range(n):
        while stack and temperatures[i] > temperatures[stack[-1]]:
            prev = stack.pop()
            answer[prev] = i - prev
        stack.append(i)

    return answer
```

### 🟢 LeetCode 496 - 下一个更大元素 I ⭐⭐

**难度**：简单 | **类型**：下一个更大元素（子集）

**题目**：`nums1` 是 `nums2` 的子集，对 `nums1` 中每个元素，求其在 `nums2` 中右边第一个更大的元素。

**思路**：先用单调栈求出 `nums2` 每个元素的下一个更大元素，存入哈希表，再查表。

```python
def nextGreaterElement(nums1, nums2):
    # 第一步：求 nums2 的下一个更大元素
    next_greater = {}
    stack = []
    for num in nums2:
        while stack and num > stack[-1]:
            next_greater[stack.pop()] = num
        stack.append(num)

    # 第二步：查表
    return [next_greater.get(num, -1) for num in nums1]
```

### 🟢 LeetCode 503 - 下一个更大元素 II ⭐⭐

**难度**：中等 | **类型**：环形数组

**思路**：遍历 2n 次，索引取模。

```python
def nextGreaterElements(nums):
    n = len(nums)
    result = [-1] * n
    stack = []

    for i in range(2 * n):
        idx = i % n
        while stack and nums[idx] > nums[stack[-1]]:
            result[stack.pop()] = nums[idx]
        stack.append(idx)

    return result
```

### 🟡 LeetCode 84 - 柱状图中最大的矩形 ⭐⭐⭐

**难度**：困难 | **类型**：柱状图 + 左右边界

**题目**：给定非负整数数组 `heights`，表示柱状图中各个柱子的高度，求其中面积最大的矩形。

**思路**：对每个柱子，用单调栈找左边和右边第一个比它矮的位置，确定宽度。

```python
def largestRectangleArea(heights):
    heights.append(0)  # 哨兵
    stack = [-1]
    max_area = 0

    for i in range(len(heights)):
        while stack[-1] != -1 and heights[i] < heights[stack[-1]]:
            h = heights[stack.pop()]
            w = i - stack[-1] - 1
            max_area = max(max_area, h * w)
        stack.append(i)

    heights.pop()
    return max_area
```

### 🟡 LeetCode 42 - 接雨水 ⭐⭐⭐

**难度**：困难 | **类型**：柱状图变体

**题目**：给定 `n` 个非负整数表示每个宽度为 1 的柱子的高度，计算能接多少雨水。

**方法一：单调栈（推荐理解思路）**

```python
def trap(height):
    stack = []
    water = 0

    for i in range(len(height)):
        while stack and height[i] > height[stack[-1]]:
            bottom = stack.pop()
            if not stack:
                break
            left = stack[-1]
            width = i - left - 1
            h = min(height[i], height[left]) - height[bottom]
            water += width * h
        stack.append(i)

    return water
```

**方法二：双指针（更优解法）**

```python
def trap(height):
    left, right = 0, len(height) - 1
    left_max, right_max = 0, 0
    water = 0

    while left < right:
        if height[left] < height[right]:
            if height[left] >= left_max:
                left_max = height[left]
            else:
                water += left_max - height[left]
            left += 1
        else:
            if height[right] >= right_max:
                right_max = height[right]
            else:
                water += right_max - height[right]
            right -= 1

    return water
```

### 🟡 LeetCode 85 - 最大矩形 ⭐⭐⭐

**难度**：困难 | **类型**：柱状图扩展到二维

**题目**：给定一个仅包含 `0` 和 `1` 的二维矩阵，找到只包含 `1` 的最大矩形。

**思路**：逐行累加形成柱状图，每行调用 LeetCode 84 的解法。

```python
def maximalRectangle(matrix):
    if not matrix:
        return 0

    rows, cols = len(matrix), len(matrix[0])
    heights = [0] * cols
    max_area = 0

    for i in range(rows):
        # 更新每列的高度
        for j in range(cols):
            heights[j] = heights[j] + 1 if matrix[i][j] == '1' else 0

        # 对当前行形成的柱状图求最大矩形
        max_area = max(max_area, largest_rectangle_area(heights))

    return max_area
```

### 🟡 LeetCode 901 - 股票价格跨度 ⭐⭐⭐

**难度**：中等 | **类型**：在线单调栈

**题目**：设计一个类，计算股票价格小于等于今天价格的最大连续天数。

**思路**：单调递减栈，栈中存储 `(价格, 跨度)` 对。

```python
class StockSpanner:
    def __init__(self):
        self.stack = []  # (price, span)

    def next(self, price):
        span = 1
        while self.stack and price >= self.stack[-1][0]:
            span += self.stack.pop()[1]
        self.stack.append((price, span))
        return span
```

### 🟡 LeetCode 316 - 去除重复字母 ⭐⭐⭐

**难度**：中等 | **类型**：单调栈 + 贪心

**题目**：给定一个字符串 `s`，去除重复字母，使每个字母只出现一次，且结果是字典序最小的。

**思路**：单调递增栈 + 计数器 + 访问标记。

```python
def removeDuplicateLetters(s):
    count = collections.Counter(s)
    in_stack = set()
    stack = []

    for ch in s:
        count[ch] -= 1
        if ch in in_stack:
            continue

        while stack and ch < stack[-1] and count[stack[-1]] > 0:
            in_stack.remove(stack.pop())
        stack.append(ch)
        in_stack.add(ch)

    return ''.join(stack)
```

```python
def removeKdigits(num, k):
    stack = []

    for digit in num:
        while k > 0 and stack and stack[-1] > digit:
            stack.pop()
            k -= 1
        stack.append(digit)

    # 如果 k 还没用完，从末尾移除
    stack = stack[:-k] if k > 0 else stack

    # 去除前导零
    result = ''.join(stack).lstrip('0')
    return result if result else '0'
```

### 🔴 LeetCode 1944 - 队列中可以看到的人数 ⭐⭐⭐

**难度**：困难 | **类型**：单调栈（从右向左）

**题目**：有 `n` 个人排成一队，每个人只能看到右边比自己矮的人，求每个人能看到的人数。

```python
def canSeePersonsCount(heights):
```

### 🔴 LeetCode 84 变体 - 子数组最小值之和 ⭐⭐⭐

> **LeetCode 907 - 子数组的最小值之和**

**题目**：给定一个整数数组 `arr`，找到所有子数组的最小值之和。

**思路**：用单调栈找到每个元素作为最小值的最左和最右边界，计算贡献。

```python
def sumSubarrayMins(arr):
    n = len(arr)
    left = [-1] * n  # 左边第一个更小的位置
    right = [n] * n   # 右边第一个更小的位置

    # 求左边第一个更小
    stack = []
    for i in range(n):
        while stack and arr[i] < arr[stack[-1]]:
            stack.pop()
        left[i] = stack[-1] if stack else -1
        stack.append(i)

    # 求右边第一个更小
    stack = []
    for i in range(n - 1, -1, -1):
        while stack and arr[i] <= arr[stack[-1]]:
            stack.pop()
        right[i] = stack[-1] if stack else n
        stack.append(i)

    # 计算贡献
    MOD = 10**9 + 7
    total = 0
    for i in range(n):
        count = (i - left[i]) * (right[i] - i)
        total = (total + arr[i] * count) % MOD

    return total
```

## 每日学习建议

### 时间安排

| 阶段 | 建议用时 | 说明 |
|------|---------|------|
| 理论学习 | 1-2 小时 | 先理解概念再看代码 |
| 纸上模拟 | 30 分钟 | 在纸上画出栈的变化过程 |
| 编码练习 | 2-3 小时 | 先手写再对照模板 |
| 刷题 | 2-3 小时 | 先独立思考至少 20 分钟 |
| 总结复习 | 30 分钟 | 记录心得和易错点 |

### 刷题方法论

1. **先独立思考** 20-30 分钟，不要急于看题解
2. **纸上模拟**：画出数组、栈的变化过程，直观理解
3. **先写伪代码**，再翻译为具体语言
4. **写完后调试**：用简单测试用例手动走一遍
5. **看题解对比**：学习更优解法和技巧
6. **隔天重做**：第二天不看题解再写一遍

### 代码模板速记

```
单调栈通用框架：
  stack = []
  for i in range(n):
      while stack and 满足弹出条件:
          top = stack.pop()
          处理弹出的元素
      stack.append(i)

下一个更大（从左到右，弹出时记录）：
  while stack and nums[i] > nums[stack[-1]]:
      result[stack.pop()] = nums[i]

左边第一个更小（入栈前记录）：
  while stack and nums[i] <= nums[stack[-1]]:
      stack.pop()
  result[i] = nums[stack[-1]] if stack else -1

柱状图最大矩形（一遍扫描 + 哨兵）：
  heights.append(0)
  stack = [-1]
  while stack[-1] != -1 and heights[i] < heights[stack[-1]]:
      h = heights[stack.pop()]
      w = i - stack[-1] - 1
      max_area = max(max_area, h * w)

环形数组：
  for i in range(2 * n):
      idx = i % n
```

## 单调栈识别指南：如何判断一道题需要用单调栈？

### 高频信号词

| 信号词 | 可能的单调栈应用 |
|--------|----------------|
| "下一个更大/更小" | 经典单调栈 |
| "第一个比...大/小的" | 单调栈找边界 |
| "柱状图"、"矩形面积" | 柱状图模板 |
| "接雨水" | 柱状图变体 |
| "字典序最小" | 单调栈 + 贪心 |
| "移除 K 个数字" | 单调栈 + 贪心 |
| "子数组的最小值之和" | 贡献法 |
| "能看到/看不到" | 从右向左单调栈 |

### 判断流程

```
题目涉及"下一个"更大/更小？
├── 是 → 单调栈
└── 否
    ├── 题目涉及"矩形面积"或"柱状图"？
    │   ├── 是 → 单调栈（柱状图模板）
    │   └── 否
    │       ├── 题目涉及"字典序最小"或"移除元素使结果最优"？
    │       │   ├── 是 → 单调栈 + 贪心
    │       │   └── 否
    │       │       ├── 题目涉及"每个元素作为最值的贡献"？
    │       │       │   ├── 是 → 单调栈 + 贡献法
    │       │       │   └── 否 → 考虑其他方法
```
