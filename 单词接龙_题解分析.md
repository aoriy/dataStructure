## 一、问题建模：从单词转换到图论

### 1.1 核心观察

**"每一对相邻的单词只差一个字母"** —— 这句话是解题的关键。

它告诉我们：如果两个单词只差一个字母，它们之间就存在一条**边**。整个问题可以建模为一个**无权无向图**，求从 `beginWord` 到 `endWord` 的**最短路径**。

### 1.2 图的构建

将每个单词视为图中的一个**节点**，如果两个单词只差一个字母，则在它们之间连一条**无向边**。

**示例**：`wordList = ["hot", "dot", "dog", "lot", "log", "cog"]`

```
        hit
         |
        hot
       /   \
     dot    lot
      |      |
     dog    log
      |      |
     cog ---+  （cog 和 log 也只差一个字母）

图的邻接关系：
  hit  → [hot]
  hot  → [dot, lot]
  dot  → [hot, dog]
  dog  → [dot, log, cog]
  lot  → [hot, log]
  log  → [dog, lot, cog]
  cog  → [dog, log]
```

### 1.3 问题转化

| 原问题 | 图论问题 |
|--------|---------|
| beginWord | 起点 |
| endWord | 终点 |
| wordList 中的单词 | 图中的节点 |
| 只差一个字母 | 两个节点间有边 |
| 最短转换序列的单词数目 | **无权图最短路径的节点数**（= 边数 + 1） |

> **为什么用 BFS？** 因为是无权图，BFS 天然保证第一次到达终点时就是最短路径。

### 1.4 判断两个单词是否只差一个字母

```python
def differ_by_one(word1, word2):
    """判断两个单词是否恰好只差一个字母"""
    if len(word1) != len(word2):
        return False
    diff_count = 0
    for c1, c2 in zip(word1, word2):
        if c1 != c2:
            diff_count += 1
            if diff_count > 1:
                return False
    return diff_count == 1
```

## 二、算法设计：三种解法

### 2.1 解法总览

| 解法 | 核心思路 | 时间复杂度 | 优化方向 |
|------|---------|-----------|---------|
| **解法一：标准 BFS** | 逐层扩展，暴力找邻居 | O(M² × N) | 简单直观 |
| **解法二：预处理优化 BFS** | 用通配符模式预处理邻接表 | O(M × N × 26) | 减少找邻居开销 |
| **解法三：双向 BFS** | 从两端同时搜索 | O(M × N × 26) / 2 | 减少搜索空间 |

> M = 单词长度，N = 单词总数

---

### 2.2 解法一：标准 BFS（暴力找邻居）

#### 思路

1. 将 `beginWord` 放入队列
2. 每次从队列取出一个单词，遍历 `wordList` 中所有单词
3. 如果某个单词与当前单词只差一个字母且未被访问，则入队
4. 第一次到达 `endWord` 时，返回当前层数

#### 图解过程

```
第 1 层：hit
第 2 层：hot          ← hit 只差一个字母的：hot
第 3 层：dot, lot     ← hot 只差一个字母的：dot, lot
第 4 层：dog, log     ← dot→dog, lot→log
第 5 层：cog          ← dog→cog（找到终点！）

路径：hit → hot → dot → dog → cog
长度：5
```

#### 伪代码

```
queue = [(beginWord, 1)]     # (单词, 层数)
visited = {beginWord}

while queue:
    word, level = queue.popleft()

    if word == endWord:
        return level

    for candidate in wordList:
        if candidate not in visited and differ_by_one(word, candidate):
            visited.add(candidate)
            queue.append((candidate, level + 1))

return 0
```

#### 缺点分析

**找邻居的效率太低**：对每个单词都要遍历整个 wordList 判断是否只差一个字母。

```
每个单词找邻居：O(N × M)    ← 遍历 N 个单词，每个比较 M 个字符
总共：O(N × N × M) = O(N² × M)
```

当 N = 5000 时，N² = 25,000,000，效率很差。

---

### 2.3 解法二：预处理优化 BFS（推荐解法）

#### 核心优化思路

> **不要遍历 wordList 找邻居，而是枚举当前单词的所有"变形"，直接查找是否存在。**

#### 关键洞察

一个长度为 M 的单词，只差一个字母的变形有多少种？

```
对于 "hot"（M=3）：
  *ot : aot, bot, cot, dot, ... hot, ... zot  → 26 种
  h*t : hat, hbt, hct, ... hot, ... hzt      → 26 种
  ho* : hoa, hob, hoc, ... hot, ... hoz      → 26 种

总共：M × 26 = 3 × 26 = 78 种变形
```

#### 预处理：构建通配符 → 单词列表的映射

```python
# 预处理
from collections import defaultdict

pattern_map = defaultdict(list)  # 通配符模式 → 匹配的单词列表

for word in wordList:
    for i in range(len(word)):
        pattern = word[:i] + '*' + word[i+1:]
        pattern_map[pattern].append(word)

# 示例：
# pattern_map["*ot"] = ["hot", "dot", "lot"]
# pattern_map["h*t"] = ["hot"]
# pattern_map["ho*"] = ["hot"]
# pattern_map["d*t"] = ["dot"]
# pattern_map["do*"] = ["dot", "dog"]
# pattern_map["*og"] = ["dog", "log", "cog"]
# ...
```

#### 找邻居的优化

```python
def get_neighbors(word):
    """通过通配符模式快速找到所有邻居"""
    neighbors = []
    for i in range(len(word)):
        pattern = word[:i] + '*' + word[i+1:]
        neighbors.extend(pattern_map[pattern])
    return neighbors
```

**优化效果**：

| 方法 | 找一个单词的邻居 | 复杂度 |
|------|----------------|--------|
| 暴力遍历 wordList | O(N × M) | 5000 × 5 = 25000 |
| 通配符模式查找 | O(M × 26) | 5 × 26 = 130 |

> 从 O(N×M) 降到 O(M×26)，当 N 远大于 26 时，提升巨大！

#### BFS 过程图解（使用预处理）

```
beginWord = "hit"

第 1 层：hit
  枚举变形：*it, h*t, hi*
  *it → 无匹配
  h*t → pattern_map["h*t"] = ["hot"] → 邻居：hot
  hi* → 无匹配
  入队：hot

第 2 层：hot
  枚举变形：*ot, h*t, ho*
  *ot → pattern_map["*ot"] = ["hot","dot","lot"] → 新邻居：dot, lot
  h*t → pattern_map["h*t"] = ["hot"] → 已访问
  ho* → pattern_map["ho*"] = ["hot"] → 已访问
  入队：dot, lot

第 3 层：dot, lot
  dot 的变形：*ot→已访问, d*t→无, do*→["dot","dog"] → 新邻居：dog
  lot 的变形：*ot→已访问, l*t→无, lo*→["lot","log"] → 新邻居：log
  入队：dog, log

第 4 层：dog, log
  dog 的变形：*og→["dog","log","cog"] → 新邻居：cog ✅ 找到终点！
  返回层数 5
```

### 2.4 解法三：双向 BFS（最优解法）

#### 核心思想

> 从 `beginWord` 和 `endWord` **同时开始 BFS**，当两端相遇时，最短路径 = 两端层数之和。

#### 为什么更快？

BFS 的搜索空间随层数**指数增长**：

```
标准 BFS（从一端）：
  第 1 层：1 个节点
  第 2 层：~26 个节点
  第 3 层：~26² = 676 个节点
  第 4 层：~26³ = 17576 个节点
  ...

双向 BFS（从两端）：
  每端只需搜索一半的层数
  第 1 层（两端各）：1 + 1 = 2 个节点
  第 2 层（两端各）：26 + 26 = 52 个节点
  总搜索空间 ≈ 标准BFS的平方根
```

#### 优化策略：始终扩展较小的队列

```python
# 每次选择节点数较少的一端进行扩展
if len(queue_begin) > len(queue_end):
    queue_begin, queue_end = queue_end, queue_begin  # 交换
```

#### 图解过程

```
begin 端：           end 端：
第 1 层：hit          第 1 层：cog
第 2 层：hot          第 2 层：dog, log
第 3 层：dot, lot     ← 相遇！dot 和 dog 只差一个字母

路径长度 = begin层数 + end层数 = 3 + 2 = 5
```

---

### 2.5 三种解法对比

| 对比维度 | 标准BFS | 预处理BFS | 双向BFS |
|---------|---------|----------|---------|
| **找邻居方式** | 遍历wordList | 通配符模式 | 通配符模式 |
| **搜索方向** | 单向 | 单向 | 双向 |
| **时间复杂度** | O(M × N²) | O(M × N × 26) | O(M × N × 26) / 2 |
| **空间复杂度** | O(N) | O(M × N) | O(M × N) |
| **实现难度** | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| **推荐程度** | 小数据可用 | ✅ 推荐 | ✅ 最优 |

## 三、完整代码实现

### 3.1 解法一：标准 BFS

```python
from collections import deque

def ladderLength(beginWord, endWord, wordList):
    word_set = set(wordList)

    # 终点不在字典中，直接返回 0
    if endWord not in word_set:
        return 0

    queue = deque([(beginWord, 1)])  # (当前单词, 层数)
    visited = {beginWord}

    while queue:
        word, level = queue.popleft()

        # 找到终点
        if word == endWord:
            return level

        # 遍历 wordList 找邻居
        for candidate in word_set:
            if candidate not in visited and differ_by_one(word, candidate):
                visited.add(candidate)
                queue.append((candidate, level + 1))

    return 0


def differ_by_one(word1, word2):
    diff = 0
    for c1, c2 in zip(word1, word2):
        if c1 != c2:
            diff += 1
            if diff > 1:
                return False
    return diff == 1
```

### 3.2 解法二：预处理优化 BFS（推荐）

```python
from collections import deque, defaultdict

def ladderLength(beginWord, endWord, wordList):
    word_set = set(wordList)

    if endWord not in word_set:
        return 0

    # 预处理：构建通配符模式 → 单词列表
    L = len(beginWord)
    pattern_map = defaultdict(list)

    for word in word_set:
        for i in range(L):
            pattern = word[:i] + '*' + word[i+1:]
            pattern_map[pattern].append(word)

    # BFS
    queue = deque([(beginWord, 1)])
    visited = {beginWord}

    while queue:
        current_word, level = queue.popleft()

        if current_word == endWord:
            return level

        # 枚举当前单词的所有通配符模式
        for i in range(L):
            pattern = current_word[:i] + '*' + current_word[i+1:]

            for neighbor in pattern_map[pattern]:
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append((neighbor, level + 1))

            # 优化：已处理的模式可以清空，避免重复访问
            pattern_map[pattern] = []

    return 0
```

### 3.3 解法三：双向 BFS（最优）

```python
from collections import deque, defaultdict

def ladderLength(beginWord, endWord, wordList):
    word_set = set(wordList)

    if endWord not in word_set:
        return 0

    # 预处理
    L = len(beginWord)
    pattern_map = defaultdict(list)

    for word in word_set:
        for i in range(L):
            pattern = word[:i] + '*' + word[i+1:]
            pattern_map[pattern].append(word)

    # 双向 BFS 初始化
    queue_begin = deque([(beginWord, 1)])  # (单词, 层数)
    queue_end = deque([(endWord, 1)])
    visited_begin = {beginWord: 1}  # 单词 → 层数
    visited_end = {endWord: 1}

    def visit(queue, visited, other_visited):
        """扩展一端的 BFS"""
        word, level = queue.popleft()

        for i in range(L):
            pattern = word[:i] + '*' + word[i+1:]

            for neighbor in pattern_map[pattern]:
                if neighbor in other_visited:
                    # 两端相遇！
                    return level + other_visited[neighbor]
                if neighbor not in visited:
                    visited[neighbor] = level + 1
                    queue.append((neighbor, level + 1))

            pattern_map[pattern] = []  # 清空已处理的模式

        return None

    while queue_begin and queue_end:
        # 始终扩展较小的队列
        if len(queue_begin) > len(queue_end):
            queue_begin, queue_end = queue_end, queue_begin
            visited_begin, visited_end = visited_end, visited_begin

        # 扩展 begin 端
        result = visit(queue_begin, visited_begin, visited_end)
        if result:
            return result

    return 0
```

### 3.4 代码逐行解析（解法二）

```python
# ──── 第1步：预处理 ────
# 对 wordList 中每个单词的每个位置，用 '*' 替换，生成通配符模式
# 例如 "dog" → "*og", "d*g", "do*"
# pattern_map["*og"] = ["dog", "log", "cog"]
# 这样查找邻居时，不需要遍历整个 wordList

# ──── 第2步：BFS ────
# 标准分层 BFS，队列存储 (单词, 层数)
# visited 集合防止重复访问

# ──── 第3步：找邻居（优化后） ────
# 对当前单词枚举 M 个通配符模式
# 通过 pattern_map 直接获取所有匹配的单词
# 时间从 O(N×M) 降到 O(M×26)

# ──── 第4步：清空已处理模式 ────
# pattern_map[pattern] = []
# 避免后续重复处理同一模式，进一步优化
```

## 四、复杂度分析

### 4.1 时间复杂度

| 步骤 | 标准BFS | 预处理BFS | 双向BFS |
|------|---------|----------|---------|
| 预处理 | 无 | O(M × N) | O(M × N) |
| BFS 找邻居 | O(N² × M) | O(M × N × 26) | O(M × N × 26) / 2 |
| **总计** | **O(M × N²)** | **O(M × N × 26)** | **O(M × N × 26)** |

> 实际上预处理BFS和双向BFS的时间复杂度量级相同，但双向BFS的常数因子更小。

**具体数据对比**（N=5000, M=5）：

| 方法 | 操作次数 |
|------|---------|
| 标准BFS | 5 × 5000² = 125,000,000 |
| 预处理BFS | 5 × 5000 × 26 = 650,000 |
| 双向BFS | ~325,000 |

> 预处理BFS比标准BFS快约 **192 倍**！

### 4.2 空间复杂度

| 数据结构 | 空间 | 说明 |
|---------|------|------|
| word_set | O(N × M) | 存储所有单词 |
| pattern_map | O(M × N) | 预处理映射 |
| queue | O(N) | BFS 队列 |
| visited | O(N × M) | 已访问单词 |
| **总计** | **O(M × N)** | |

## 六、扩展思考

### 6.1 如果需要输出具体路径怎么办？

**LeetCode 126 — 单词接龙 II**

在 BFS 的基础上，需要记录每个单词的**前驱单词**，然后从 endWord 回溯到 beginWord。

```python
# 在 BFS 中记录前驱
parent = {}  # 单词 → 前驱单词列表
parent[beginWord] = []

# 当发现 neighbor 时：
parent[neighbor] = parent.get(neighbor, []) + [current_word]

# BFS 结束后，从 endWord 回溯所有路径
def find_all_paths(parent, beginWord, endWord):
    if endWord == beginWord:
        return [[beginWord]]
    paths = []
    for prev in parent[endWord]:
        for path in find_all_paths(parent, beginWord, prev):
            paths.append(path + [endWord])
    return paths
```

> **注意**：由于可能有多条最短路径，每个单词可能有多个前驱，需要用列表存储。

### 6.2 如果单词长度很长怎么办？

当 M 很大时，枚举 M × 26 个变形仍然高效，因为 26 是常数。

但如果 M 极大（如 M = 1000），可以考虑：
- **位运算优化**：用位掩码表示字母差异
- **Trie 树**：按前缀组织单词，减少搜索范围

### 6.3 如果 wordList 非常大（百万级）怎么办？

- **预处理存储在外部**：pattern_map 可能很大，考虑分片存储
- **过滤无关单词**：先筛选与 beginWord/endWord 长度相同的单词
- **A* 搜索**：用启发式函数（如与 endWord 的差异字母数）优先扩展更接近终点的节点

### 6.4 这道题的考察点总结

| 考察点 | 对应知识点 |

### 6.5 解题模板速记

```
单词接龙标准解法：
  1. 检查 endWord 是否在 wordList 中
  2. 预处理：构建 pattern_map[通配符] = [单词列表]
  3. BFS：
     queue = [(beginWord, 1)]
     while queue:

         if word == endWord: return level

                     queue.append((neighbor, level + 1))
  4. 返回 0
```
